# Telegram API 凭证安全说明

## 🔐 使用公共 API 凭证安全吗？

**简短回答**: ✅ **安全**，不会对您的账号造成风险。

## 📖 详细解释

### 1. API 凭证的作用

`TG_API_ID` 和 `TG_API_HASH` 只是用于：
- ✅ 标识应用程序（告诉 Telegram 这是哪个应用）
- ✅ 建立连接到 Telegram 服务器

它们**不能**：
- ❌ 访问您的账号
- ❌ 读取您的消息
- ❌ 冒充您的身份

### 2. 真正的安全凭证

您的账号安全由以下保护：

#### Session 文件（最重要）
- 存储在服务器的 `/data` 目录
- 包含您的登录状态
- **只有拥有 session 文件的人才能访问您的账号**

#### 手机验证码
- 每次登录都需要手机验证码
- 验证码发送到您的手机
- 没有验证码无法登录

#### 2FA 密码（如果启用）
- 额外的安全层
- 只有您知道密码

### 3. 类比说明

把 Telegram 登录想象成进入一个大楼：

```
TG_API_ID/HASH = 大楼的地址（公开信息）
手机验证码 = 门禁卡（只有您有）
Session 文件 = 房间钥匙（登录后获得）
2FA 密码 = 保险箱密码（可选）
```

- **API 凭证** 只是告诉您"大楼在哪里"
- 但要进入您的"房间"，需要**您的**门禁卡和钥匙

### 4. 为什么原项目使用公共凭证？

原项目（tg-signer）使用公共 API 凭证是因为：

1. **方便用户** - 不需要每个用户都去申请 API 凭证
2. **开源项目常见做法** - 很多 Telegram 工具都这样做
3. **安全** - Session 文件才是真正的安全凭证

### 5. 公共 vs 私有 API 凭证

| 特性 | 公共凭证 | 私有凭证 |
|------|---------|---------|
| 安全性 | ✅ 相同 | ✅ 相同 |
| 方便性 | ✅ 无需申请 | ❌ 需要申请 |
| 速率限制 | ⚠️ 共享限制 | ✅ 独立限制 |
| 推荐场景 | 个人使用 | 大规模使用 |

## ⚠️ 真正的安全风险

### ✅ 应该担心的

1. **Session 文件泄露**
   - 如果别人获得您的 session 文件，可以访问您的账号
   - **解决方案**: 保护好服务器，不要分享 session 文件

2. **服务器被入侵**
   - 如果服务器被黑客入侵，session 文件可能被盗
   - **解决方案**: 使用强密码，启用 2FA，定期更新

3. **数据目录权限**
   - 如果其他人能访问 `/data` 目录
   - **解决方案**: 正确设置文件权限

### ❌ 不需要担心的

1. ❌ 使用公共 API 凭证
2. ❌ API 凭证被其他人知道
3. ❌ 多人使用相同的 API 凭证

## 🔒 最佳安全实践

### 1. 保护 Session 文件

```bash
# 确保数据目录只有应用能访问
chmod 700 /data
```

### 2. 启用 2FA

在 Telegram 设置中启用两步验证：
1. 打开 Telegram
2. 设置 → 隐私与安全 → 两步验证
3. 设置密码

### 3. 定期检查活动会话

在 Telegram 中：
1. 设置 → 隐私与安全 → 活动会话
2. 检查是否有可疑登录
3. 终止不认识的会话

### 4. 使用强密码

- Web UI 管理员密码要强
- 2FA 密码要强
- 不要重复使用密码

## 💡 何时使用私有 API 凭证？

建议申请自己的 API 凭证，如果：

1. **大规模使用**
   - 管理超过 10 个账号
   - 频繁发送消息
   - 避免共享速率限制

2. **商业用途**
   - 提供服务给其他人
   - 需要稳定的 API 配额

3. **隐私考虑**
   - 不想与其他人共享 API 凭证
   - 需要独立的使用统计

### 如何申请

1. 访问 https://my.telegram.org
2. 登录您的 Telegram 账号
3. 点击 "API development tools"
4. 填写应用信息
5. 获取 API ID 和 API Hash
6. 在 Zeabur 设置环境变量

## 📊 总结

| 问题 | 答案 |
|------|------|
| 使用公共 API 凭证安全吗？ | ✅ 安全 |
| 会泄露我的账号吗？ | ❌ 不会 |
| 需要申请自己的凭证吗？ | ⚠️ 个人使用不需要 |
| 真正的安全凭证是什么？ | ✅ Session 文件 + 验证码 + 2FA |

**结论**: 使用原项目的公共 API 凭证是**完全安全**的。真正需要保护的是您的 **Session 文件**和**服务器访问权限**。
